ShowWork

An AI vulnerability review for assignments

Why this tool exists −

In the past, our assignments were largely about output, and the process a student used to get there was usually evident in that output. A clear, well-reasoned plan implied a clear, well-reasoned process behind it. Now that students have powerful tools to help produce that output, the old assumption breaks down. A polished deliverable no longer tells us, on its own, how the student got there, or whether the competencies we are trying to assess were exercised.

A familiar reframe: Early in my career, I worked with children with special needs, and I kept a reminder on my bulletin board: “Don’t ask if this child is smart, ask how this child is smart.” A similar shift applies here. The question is not whether a student used AI, but how do we help this student demonstrate the specific competencies this assignment is meant to reveal.

This reframe changes the academic integrity conversation too. Reasoning-visible assignments shift the central question from “did this student use AI?” (faculty proving an absence) to “did this student demonstrate the competency?” (faculty assessing visible reasoning).

ShowWork helps you apply that question to a real assignment. Choose your assignment type, paste in the prompt and rubric (and optionally the competency it should demonstrate), and ShowWork checks it against known AI-vulnerability patterns relevant to that type. It returns a summary plus a card for each pattern checked: worth revising, partial, or holding up, each with a plain-language explanation and specific options to consider. None of this requires rewriting the assignment from scratch; the goal is to find the gap between what the assignment claims to measure and what it actually asks students to do, and to close it with a small, targeted change.

ShowWork was based on the work by Jean Gordon, Eric Richardson, Dawn Oetjen, Reid Oetjen, and Nikki Strawn, from the AUPHA Art of Teaching Institute course “Future Fluent by Design: Reimagining Assessments for the Age of AI” (Medical University of South Carolina and University of Central Florida).

How to use this tool −
  1. Choose a review method. Use your own AI account (Claude, ChatGPT, or Gemini) to run the review directly here, or copy a ready-made prompt to use in whatever AI tool you already have. Note: formatted ShowWork cards work best when using Claude. If you use ChatGPT with the copy prompt, type "Make this output reader friendly" in ChatGPT to get a readable result there.
  2. Select your assignment type. This determines which diagnostic patterns ShowWork checks, so the review stays focused on what matters most for that kind of deliverable.
  3. Paste in your assignment and rubric. Use the assignment prompt as students see it, and the rubric or grading criteria you use to evaluate it. Naming the competency the assignment should demonstrate is optional but helps sharpen the review.
  4. Run the review. ShowWork returns an overall summary and a card for each pattern checked, showing the vulnerability, why it matters, and a few options to consider for addressing it.
  5. Use the prompts if helpful. Each card includes a brainstorm prompt you can copy into any AI tool to explore revision ideas, and a check prompt you can use after revising, to see whether the gap has actually closed.
  6. This works on drafts too. If you are still designing a course, running an early outline through ShowWork can help shape the assignment from the start.
